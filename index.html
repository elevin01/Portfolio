Detecting Inconsistencies in the Modeling of SNOMED CT

Role: Researcher and Developer

Summary: This project aimed to identify and address inconsistencies in SNOMED CT, the world's most sophisticated clinical terminology database, using machine learning, lexical similarity analysis, and advanced database management techniques.

  • Designed and implemented a methodology to identify inconsistencies in SNOMED CT by leveraging sibling concepts and attribute modeling.
  • Developed and utilized a Word2Vec-based lexical similarity algorithm to calculate semantic similarities between sibling terms, achieving a similarity score threshold of 0.9.
  • Created database pipelines to manage and query SNOMED CT concepts efficiently, ensuring seamless integration of term data for analysis.
  • Automated data processing and validation using Python scripts:
    • Data Retrieval: Extracted and processed concepts based on tags such as "disorder," "procedure," and "findings."
    • Database Management: Implemented robust functions to validate files, populate tables, and query attribute counts and relationships.
    • Inconsistency Detection: Flagged concepts with attribute discrepancies for manual audits, significantly reducing the workload for experts.
  • Results:
    • Identified 70 sets of lexically similar terms from 150 input terms, with 20 flagged for inconsistencies.
    • Demonstrated the potential of computational techniques to streamline manual audits and enhance SNOMED CT's accuracy.
